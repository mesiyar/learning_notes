# 当主从复制过程中,从服务器的数据没有及时同步到时,有哪些解决方案.
* 场景
```
Q:当一个用户注册了,主服务器数据写入成功,但是从服务器数据还尚未同步成功时.该用户又要登录到应用,
应该当如何解决?
A:可以考虑在用户数据注册成功之后,将用户数据写入到redis服务器中,若是登录时在从服务器中没有查询
到对应的数据,可以到redis服务器中查询.
Q:当遇到了主从复制不一致时,有什么解决方案?
A:1.从服务器的硬件配置应当选择比主服务器更高的配置
  2.可以适当的将从服务器的安全等级降低.具体的配置
  3.版本应当选择5.7以上版本,5.7之前执行同步的线程只有一个,5.7之后增加到了多个
  4.从库的模式设置成只读模式;set global read_only=1; 
```
